# Personal Wiki - 个人知识库技能

**版本**: v1.0.0
**作者**: 张家诚 (诚)
**描述**: 基于 Karpathy LLM Wiki 理念的个人知识管理技能，支持碎片化学习、自动整理、智能问答

---

## 🎯 什么时候使用

当用户想要：
1. **建立个人知识库** - 从零搭建结构化 Wiki
2. **碎片化学习** - 小红书收藏→自动整理→持续积累
3. **知识问答** - 对知识库提问，获取综合答案
4. **健康检查** - 定期扫描知识库，发现矛盾/过时/孤立内容
5. **生成报告** - 周报/学习总结/知识图谱

## 🏗️ 核心理念

基于 **Andrej Karpathy** 的 LLM Wiki 方法论：

> 不要把 LLM 当搜索引擎用，而是让它像图书管理员一样，帮你持续维护一个 Markdown 知识库。

### 与传统 RAG 的区别

| 维度 | 传统 RAG | Personal Wiki |
|------|----------|---------------|
| 知识存储 | 原始文档 + 向量索引 | 结构化 Wiki + 交叉引用 |
| 查询方式 | 临时检索 | 先编译再查询 |
| 知识积累 | 无（每次重新检索） | 持续积累（越用越聪明） |
| 维护成本 | 低（但知识不沉淀） | 低（LLM 自动维护） |

## 📁 目录结构

```
my-wiki/
├── raw/                    # 原始资料（只读）
│   ├── inbox/              # 待处理
│   └── archive/            # 已归档
│       ├── articles/
│       ├── videos/
│       └── notes/
├── wiki/
│   ├── index.md            # 总目录
│   ├── log.md              # 操作日志
│   ├── topics/             # 主题页
│   ├── sources/            # 来源页
│   ├── concepts/           # 概念页
│   └── queue/              # 待学习队列
├── outputs/                # 生成的报告
└── AGENTS.md               # 知识库 Schema
```

## ⚙️ 核心操作

### 1. Ingest（摄入）

**触发**: 用户说"处理 inbox"或"收集 XX 内容"

**流程**:
```
1. 读取 raw/inbox/ 中的文件
2. 对每个文件:
   - 提取元数据（标题、来源、类型）
   - 生成 3-5 个核心观点
   - 分配标签（领域 + 难度 + 优先级）
   - 创建来源页 wiki/sources/{slug}.md
   - 更新相关主题页
   - 移动到 raw/archive/{type}/
   - 追加到 wiki/log.md
   - 更新 wiki/index.md
3. P0/P1 优先级的内容 → 立即推送摘要
```

**示例**:
```
用户：处理 inbox
助手：✅ 已处理 5 篇文章
- AI 产品：3 篇（P1）
- 投资：1 篇（P2）
- 生活：1 篇（P3）
已创建 wiki 页面，高优先级内容已推送
```

---

### 2. Query（问答）

**触发**: 用户对知识库提问

**流程**:
```
1. 搜索 wiki/index.md 找到相关页面
2. 读取相关主题页/概念页/来源页
3. 综合回答 + 引用来源
4. 答案可存回 wiki/outputs/
```

**示例**:
```
用户：Karpathy 的知识库方法是什么？
助手：根据知识库中的 3 篇 Karpathy 相关文章：

1. 核心理念：让 LLM 持续维护 Markdown 知识库
2. 三层架构：raw/ + wiki/ + AGENTS.md
3. 三个操作：Ingest → Query → Lint

来源：
- [[Karpathy 亲自教你搭建知识库]]
- [[Karpathy 分享个人知识库思路]]
- [[Karpathy AI 知识库经验分享]]
```

---

### 3. Lint（健康检查）

**触发**: 用户说"检查 wiki 健康"或定时任务

**流程**:
```
1. 扫描 wiki/ 所有页面
2. 检查:
   - 矛盾内容（不同页面说法冲突）
   - 过时信息（新资料推翻旧结论）
   - 孤立页面（无入链）
   - 缺失交叉引用
   - 重要概念无独立页面
3. 生成修复建议
4. 可选：自动修复
```

**示例**:
```
用户：检查 wiki 健康
助手：📊 Wiki 健康报告

✅ 好的方面:
- 来源页格式统一
- 索引更新及时

⚠️ 需要修复:
- 3 个孤立页面（无入链）
- 2 处矛盾（GC 含量描述不一致）
- 5 个概念缺少独立页面

建议操作:
1. 为"剂量 - 反应模型"创建概念页
2. 统一 GC 含量描述
3. 添加孤立页面的入链
```

---

### 4. Export（导出）

**触发**: 用户说"生成周报"或"本周学习总结"

**流程**:
```
1. 读取 wiki/log.md 中指定时间段记录
2. 按领域分类统计
3. 生成结构化报告
4. 可选：推送/保存到 outputs/
```

**示例**:
```
用户：生成周报
助手：📋 本周学习报告 (2026-04-01 ~ 04-07)

📊 统计:
- 新增来源：12 篇
- AI 产品：7 篇
- 投资：2 篇
- 研究：1 篇
- 工具：1 篇
- 生活：1 篇

🎯 高优先级内容:
1. Karpathy LLM Wiki 教程（3 篇系列）
2. AI 产品求职资源（3 篇）
3. Python 量化入门

📝 待学习:
- 2 篇 P1 内容待深入
```

## 🔧 配置说明

### AGENTS.md 模板

```markdown
# 知识库 Schema

## 这是什么
一个关于 [主题] 的个人知识库

## 目录结构
- raw/ 原始资料（只读）
- wiki/ 结构化知识（LLM 维护）
- outputs/ 生成的报告

## 标签规范
- 领域：#ai-product #investing #research #tools #life
- 难度：#beginner #intermediate #advanced
- 类型：#tutorial #news #opinion #data

## 优先级规则
- P0: 紧急重要 → 立即推送
- P1: 重要不紧急 → 每日摘要
- P2: 一般 → 每周报告
- P3: 参考材料 → 仅归档
```

## 🎯 设计原则

1. **简单优先** - 不需要向量数据库，index.md 就够了
2. **LLM 维护** - 所有脏活累活 LLM 做
3. **持续积累** - 每次使用都让知识库更丰富
4. **用户友好** - 自然语言交互，无需记命令
5. **专注小红书** - 先做好一个平台，再扩展其他
6. **渠道可选** - 推送渠道让用户自己选
