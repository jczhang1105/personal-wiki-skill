# Personal Wiki Skill

📚 **基于 Karpathy LLM Wiki 理念的个人知识管理技能**

> 不要把 LLM 当搜索引擎用，而是让它像图书管理员一样，帮你持续维护一个 Markdown 知识库。

---

## ✨ 功能特性

| 功能 | 说明 | 状态 |
|------|------|------|
| **Ingest** | 摄入资料→总结→归档 | ✅ |
| **Query** | 搜索 Wiki→综合回答 | ✅ |
| **Lint** | 健康检查→修复建议 | ✅ |
| **Export** | 生成周报/学习总结 | ✅ |
| **小红书收集** | 自动收集收藏 | ✅ |
| **自动分类** | 智能标签 + 优先级 | ✅ |

## 🚀 快速开始

### 安装

```bash
npx clawhub@latest install personal-wiki --force
```

### 第 1 步：初始化知识库

```
用户：帮我搭建一个 AI 产品学习的知识库

助手：好的！正在创建知识库结构...
✅ 已创建目录和核心文件
现在可以开始添加内容了！
```

### 第 2 步：配置小红书收集

```
用户：配置小红书收藏

助手：📕 配置小红书收集：
1. 在浏览器中打开小红书并登录
2. 打开浏览器开发者工具 (F12)
3. 复制 Cookie 中的 xsec_token
4. 粘贴到这里

或者使用简化流程：
- 每天定时检查你的小红书收藏
- 自动保存到 inbox 并处理
```

### 第 3 步：配置推送渠道（可选）

```
用户：配置推送渠道

助手：📤 选择推送渠道：
1. 微信 - 关注 OpenClaw 助手
2. Telegram - 创建 Bot
3. Discord - 创建 Bot
4. 邮件 - 配置 SMTP
5. WebChat - 无需配置（默认）

回复数字选择 (1-5)
```

### 日常使用

```
用户：处理 inbox
助手：✅ 已处理 5 篇文章，已创建 wiki 页面

用户：我收藏了哪些 AI 产品求职资源？
助手：根据你的知识库，有 6 篇相关资源...

用户：检查 wiki 健康
助手：📊 Wiki 健康报告：3 个孤立页面，2 处矛盾...

用户：生成周报
助手：📋 本周学习报告：新增 12 篇，AI 产品 7 篇...
```

## 📁 目录结构

```
my-wiki/
├── raw/                    # 原始资料（只读）
│   ├── inbox/              # 待处理
│   └── archive/            # 已归档
├── wiki/
│   ├── index.md            # 总目录
│   ├── log.md              # 操作日志
│   ├── topics/             # 主题页
│   ├── sources/            # 来源页
│   ├── concepts/           # 概念页
│   └── queue/              # 待学习队列
├── outputs/                # 生成的报告
└── AGENTS.md               # 知识库 Schema
```

## 📊 核心理念

### 传统 RAG vs Personal Wiki

| 维度 | 传统 RAG | Personal Wiki |
|------|----------|---------------|
| 知识存储 | 原始文档 + 向量索引 | 结构化 Wiki + 交叉引用 |
| 查询方式 | 临时检索 | 先编译再查询 |
| 知识积累 | 无（每次重新检索） | 持续积累（越用越聪明） |

### 为什么有效？

- **LLM 维护** - 不会厌倦更新交叉引用
- **持续积累** - 每次使用都让知识库更丰富
- **简单优先** - 不需要向量数据库，index.md 就够了

## 🛠️ 技术实现

### 核心脚本

| 脚本 | 功能 |
|------|------|
| `scripts/init_wiki.py` | 初始化知识库 |
| `scripts/ingest.py` | 摄入处理 |
| `scripts/query.py` | 搜索问答 |
| `scripts/lint.py` | 健康检查 |
| `scripts/export.py` | 导出报告 |

### 定时任务

```json
{
  "每日收藏收集": "0 20 * * *",
  "每日自动处理": "30 20 * * *",
  "每周健康检查": "0 10 * * 0",
  "每周学习报告": "0 18 * * 5"
}
```

- [[相关主题]]
